# 落地手艺与反模式

归宿判对了只是第一步,写进目标资产时还有一套手艺——套用既有文风、不引入新论断、不制造重复。下面按资产
类型给落地手艺,末尾是一张跨四类共享的 Rationalization Table。

## 纪律文档

- **先读目标文档的既有结构再落笔**——它已经有的章节划分、条目格式(是"一句话+一句 why"、还是带编号的
  规则列表)、语言风格,照套,不要另起一套自己觉得更好的格式。
- **条目必须带 why**——只写"不要做 X"留不住,要写"不要做 X,因为(这条证据指向的具体后果)"。why 越
  具体,这条经验越经得起未来被质疑时的复核。
- **合并同落点的多条候选**,不要在同一份文档里分别追加造成两段说同一件事——如果两条候选经验最终都要
  落进同一节,先合并成一次连贯增补再落笔。

## skill

- **先判断是塞进既有 skill 的一节,还是真的需要一个新 skill**——如果目标项目已有一个高度相关的 skill,
  优先考虑作为它的一节增补;新增一个与既有 skill 高度重叠的独立 skill 会造成两份主文件漂移,谁都不知道
  该信哪份。
- **触发描述要精确**——skill 的价值取决于它能不能在恰当的时候被找到;笼统或过宽的触发描述会让它要么
  永远沉默、要么被无关任务误触发。
- **纪律型内容(压力下会被合理化掉的规则)要有 Rationalization Table**——只写"应该怎么做"不够,要写
  "你会怎么对自己说 / 现实是什么",否则这条规则在真正需要它的高压时刻大概率会被绕过。这类 skill 落地
  前应该先设想一下:没有这段文字时,agent 在赶时间/有理由偷懒的情况下会怎么错——错的地方就是该写进
  Rationalization Table 的那一行。

## workflow

- **只固化结构,不固化判断**——脚本里该出现的是"先做 A、再并行做 B 和 C、汇总后做 D"这类形状,不该
  出现"如果情况像这样就这么做、否则那么做"这类判断分支;判断分支是判断力,应该留给触发这段 workflow
  的 agent,而不是塞进脚本假装是确定性的。
- **落地前先确认这条经验真的是"每次都一样"的机制**,而不是"目前只见过一次的模式"——机制形状的证据
  门槛比事实/判断力更高,因为固化错了返工成本也更高(要拆迁一整段编排脚本)。

## subagent

- **先确认这个角色会被反复以同一身份召唤**——如果目标项目里这类任务只出现过一次,新建一个专职角色定义
  是过度工程,普通一次性派发就够了。
- **persona 的工具/上下文边界要写清楚**——专职角色的价值就在于它只从一个固定视角、带一套固定工具边界
  工作,边界含糊的"角色定义"和普通派发没有本质区别。

## Rationalization Table —— 四类资产共享的自我说服清单

| 你会对自己说 | 现实 |
|---|---|
| "这条经验挺重要的,得进 skill 才够重" | 重要 ≠ 可复用判断力。重要的一次性事实老实进纪律文档更合适——skill 是给"要被反复触发调用的判断力"用的机制,塞进一次性事实只会让它的触发描述变得难以精确触发。 |
| "顺手都塞进纪律文档得了,反正它已经很长了" | 判断力类内容塞进纪律文档会让文档失去线性可读性,且下次没人会在压力下去纪律文档里翻一段判断力——纪律文档适合"读一遍记住"的事实,不适合"每次用到都要重新翻查"的方法论。 |
| "证据不够具体,先按经验直觉把措辞写通用一点" | 违反本 skill 的唯一硬约束:不允许脱离证据泛化。措辞不够具体时应该**收窄**而不是**泛化**——写"在某场景下观察到某结果"而非"总是应该怎样"。细则见 [evidence-fidelity.md](evidence-fidelity.md)。 |
| "这个项目没有对应的基础设施,就别提这条经验了" | 违反"绝不因为没地方放就放弃整条经验"——降级为纪律文档指针,不蒸发。 |
| "这条经验和某个已有 skill 有点像,干脆另起一个新 skill 更干净" | 先检查能否作为该已有 skill 的一节增补——新增一个与既有 skill 高度重叠的 skill 会造成两份主文件漂移。 |
| "这条候选经验证据太弱/太模糊,判断不出归宿,干脆不处理了" | **绝不静默丢弃**——证据弱不是不处理的理由,是"标注置信度低、按纪律文档托底落地"的理由。哪怕最终这条经验被人工审阅否决,它也应该先被看见,而不是在规划阶段就悄悄消失。 |
| "这个模式只见过一次,但感觉挺通用的,直接固化成 workflow/skill 好了" | 单一实例还不能确认是"确定性机制"或"可复用判断力"——证据只支持"观察到一次",落地时按这个证据强度收窄措辞(如"这一次…"),归宿也应更保守(默认纪律文档式的观察记录),而非一步到位固化成结构或独立判断力文档。 |
