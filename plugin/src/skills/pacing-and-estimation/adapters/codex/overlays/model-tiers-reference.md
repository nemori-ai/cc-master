## Codex 模型档位（provider mapping 尚未编码）

不要套用 Claude 或 Cursor 的 model tier 表。Codex / OpenAI 的模型、reasoning effort、Fast tier 与 credit 消耗需要单独 provider mapping；**本 adapter 当前没有编码完整分档表**。

Codex 官方按档给出了明确的 credit 消耗梯度——旗舰档 : 主力档 : 轻量档 ≈ **125 : 62.5 : 18.75**（每 MTok 输入 credit）。换算下来，同样 token 量下主力档约为旗舰档的一半消耗、轻量档约为旗舰档的 1/6.7——**档位选择直接映射配额消耗倍率**，选档前先按这个倍率估一下这次要花多少配额。

在完整 mapping 落地前，按任务轴选档（不按价格单调排序）：

- 简单读扫、检索、格式化、批量机械改动：轻量档。
- 常规实现 / 调研摘要：主力档。
- 复杂设计、跨文件推理、有状态实现：高能力档。
- 临界路径与不可逆决策：优先质量；配额吃紧时先降前两类，不降这一类。
- 终端 / agentic 执行：主力档常不输旗舰，别默认为它加钱。

主线固定一个模型；省钱靠给 leaf 分档，不靠中途切主会话模型。
