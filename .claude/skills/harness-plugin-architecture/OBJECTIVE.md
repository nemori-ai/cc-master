# OBJECTIVE — harness-plugin-architecture

J_top: 维护者为 N 个 origin harness 保持单一 canonical plugin 语义，以 host-native Track A/B 合同兑现可测试的能力等价，并把 headless runtime 移交唯一 owner。
baseline_reference:
  user_task: 新增或修订一个 host adapter，或把 cross-harness capability 接入 origin hooks/commands/skills/native attempt surface。
  without_skill_floor: 维护者按事件名或最近邻 host 复制实现，把绿色 dist 当行为等价；跨 surface gap 没有 Capability Card/CONTRACT，或把 same/other-harness CLI runtime 塞回 plugin adapter。
  expected_uplift: 维护者先按 fact→intent/CONTRACT→Track A/B equivalence→host-native adapter/projection/probe 推进，并守住 origin/runtime owner boundary。
strict_dims: [canonical-single-SSOT, origin-capability-equivalence-and-owner-boundary]
rationale: 第一维防 canonical/adapter fork；第二维同时锁住用户可见能力等价与 origin plugin / headless runtime 的 bounded-context。具体 host mechanism、provider facts与措辞保留演化空间。
