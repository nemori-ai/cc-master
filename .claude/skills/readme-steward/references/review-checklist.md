## 事实诚实

- [ ] 新增或改写的能力 claim 能指向代码、feature manual、ADR、脚本或现有事实源。
- [ ] 没有把愿景写成已落地能力。
- [ ] 没有绝对化承诺预算、切号、不中断、完成质量等高风险能力。
- [ ] 如果能力状态复杂，README 链接 feature manual，而不是复制一份状态矩阵。

## 入口清晰

- [ ] 首屏 15 秒内能回答：cc-master 是什么、给谁、解决什么痛点。
- [ ] 最短启动命令在首屏或很靠前的位置。
- [ ] “什么时候别用”仍然存在且清楚。
- [ ] 内部架构术语没有抢在用户价值之前。

## 双语一致

- [ ] `README.md` 和 `README_zh.md` 的命令、版本、路径、链接一致。
- [ ] 能力边界一致；中文没有额外夸大，英文没有遗漏关键限制。
- [ ] 图片路径分别使用对应语言资产时，文件存在。
- [ ] 语言风格自然，不是机械逐句翻译。

## 可运行路径

- [ ] 安装命令可复制，没有过期 tag、路径或参数。
- [ ] pin plugin / ccm version 的说明符合当前 installer 行为。
- [ ] harness 参数说明和当前支持面一致。
- [ ] README 中出现的命令不依赖未说明的 cwd 或环境。

## 贡献者路径

- [ ] 明确提示改 `plugin/src`，不要手改 `plugin/dist`。
- [ ] 保留 projection / sync check / git hook 的最小命令。
- [ ] 保留 `.claude/skills` → `.agents/skills` 同步命令。
- [ ] 深层贡献者规则链接到 AGENTS.md 或相关文档，不在 README 复述红线正文。

## 形式和可读性

- [ ] 没有 badge soup、emoji soup 或泛 marketing 形容词。
- [ ] 列表是为了扫描，不是为了堆 feature。
- [ ] 截图 / 图片说明真实产品状态，不是抽象装饰。
- [ ] Markdown 链接有效，图片 alt text 有意义。
- [ ] 局部改动没有无意中重写整篇语气和叙事。
