  触发"）；本文档撰写过程中对同一块 board 的一次 `ccm board show ... 2>&1 | head` 只读命令的独立复现

- **建议归宿类型**：纪律文档
- **建议落点**：`scripts/sync-plugin-dist.sh` 的脚本内注释,或 `plugin/src/AGENTS.md` 的常用命令说明附近
  （待后续判断,这条更偏"操作提示"而非需要触发式召回的判断力,倾向纪律文档而非 skill）
- **证据**：board 日志条目 `08:52:08Z`（"工作树 dist/codex 被并行 sync 半途清空——dist 为纯生成物,端点
  统一重投影修复"）
- **候选内容草稿**：多个并行 agent 同时对同一 `plugin/dist/<host>` 目录跑 `sync-plugin-dist.sh` 时可能
  发生工作树竞态（一个 agent 的 `rm -rf` 与另一个的写入交叉,导致目录一度不完整）。这不需要靠加锁互斥
  解决——`plugin/dist/` 是从 `plugin/src/` 机械投影出的纯生成物,端点收口前统一重新跑一次投影脚本即可
  恢复一致状态,不需要人工排查"丢了哪些文件"。
